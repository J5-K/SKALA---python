# 실습 5 Polars · DuckDB 성능 비교

Pandas: CSV를 메모리에 읽고 groupby()로 집계
Polars: scan_csv()로 Lazy API 사용
DuckDB: CSV 파일을 DataFrame으로 미리 변환하지 않고 SQL에서 직접 읽음

# 개선사항/느낀점
- 현재 코드는 각 엔진에서 csv 읽기와 집계에 걸린 전체 시간을 한 번씩 측정함
- 실행 환경에 따른 편차를 줄이기 위해 향후 각 엔진을 여러 번 실행하고 평균과 중앙값 비교 가능
- 데이터 크기를 단계별로 설정해 규모에 따라 엔진별 성능 차이가 어떻게 달라지는지 확인 가능
- 같은 데이터를 같은 기준으로 집계하더라도 사용하는 데이터 처리 엔진에 따라 실행 시간이 달라질 수 있다는 것을 확인
- 단순히 실행 시간이 빠른 엔진을 선택하는 것보다 데이터 크기, 메모리 사용량, SQL 사용 여부, 코드의 가독성을 함께 고려해야 함
- 실제로 결과 값이 같은지 검증하는 과정이 반드시 필요할 듯
