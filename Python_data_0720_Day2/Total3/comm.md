# 종합실습 3 분석 자동화 및 리포트 생성

종합설명
    - sales_raw.csv를 읽고 필요한 데이터를 정제한 뒤 매출 KPI와 표가 포함된 HTML 리포트를 자동으로 생성하는 과제
    - 한 번 실행하는 것에서 끝나지 않고 일정한 간격이나 지정된 시간마다 반복 실행할 수 있도록 구성
    - 설정, 데이터 처리, HTML 디자인, 실행 스케줄을 각각 분리해서 수정하기 쉬운 구조로 만듦

파일별 역할
    - config.py: 리포트 제목, 데이터 위치, 템플릿 위치, output 위치, 상위 지역 개수를 관리
    - report.py: 데이터 정제, KPI 집계, Jinja2 렌더링, HTML 파일 생성을 담당
    - run_scheduler.py: 한 번 실행, 초 단위 반복 실행, 매일 지정 시각 실행을 담당
    - templates/report.html: HTML 화면 디자인을 담당
    - output/: 실행할 때마다 생성되는 HTML 리포트를 저장

step0 설정 분리
    - Config를 dataclass(frozen=True)로 만들어 실행 도중 경로나 제목이 임의로 바뀌지 않도록 함
    - 코드 여러 곳에 경로를 직접 적지 않고 CONFIG를 통해 같은 값을 공유함

step1 데이터 정제
    - quantity, unit_price, discount를 숫자로 변환
    - 0 이하의 가격은 잘못된 값으로 보고 결측치로 바꾼 뒤 카테고리별 중앙값으로 채움
    - region 결측치는 Unknown으로 표시
    - 수량 × 가격 × (1 - 할인율)로 sales_amount를 계산

step2 KPI와 매출표 집계
    - 전체 주문 수, 전체 판매 수량, 총매출, 평균 주문액을 KPI로 계산
    - 카테고리별 주문 수·판매 수량·매출을 표로 생성
    - 지역별 매출을 계산하고 상위 지역만 리포트에 표시

step3 Jinja2 HTML 렌더링
    - Python 문자열로 HTML을 길게 조립하지 않고 report.html 템플릿을 별도로 사용
    - Python은 계산을 담당하고 HTML은 화면 구성을 담당하도록 역할을 나눔
    - output 폴더가 없으면 자동으로 생성하도록 함

step4 타임스탬프 파일 저장
    - report_연월일_시분초.html 형식으로 저장해서 실행할 때마다 이전 결과를 덮어쓰지 않도록 함
    - 리포트 안에도 생성 시각을 표시해 언제 만든 결과인지 확인 가능

step5 실행 자동화
    - report.py를 실행하면 즉시 한 번 리포트를 생성
    - --interval 60을 사용하면 60초마다 새 리포트를 생성
    - --daily 09:00을 사용하면 매일 오전 9시에 실행
    - 어떤 실행 방식을 사용하더라도 같은 run_once()를 호출해 결과가 달라지지 않도록 함

실행
    - .venv/bin/python Python_data_0720_Day2/Total3/report.py

반복 실행
    - .venv/bin/python Python_data_0720_Day2/Total3/run_scheduler.py --interval 60

매일 지정 시각 실행
    - .venv/bin/python Python_data_0720_Day2/Total3/run_scheduler.py --daily 09:00

cron 예시
    - 0 9 * * * cd /Users/juoh/workspace/skala_python && .venv/bin/python Python_data_0720_Day2/Total3/report.py

성공 신호
    - output 폴더에 타임스탬프가 포함된 HTML 파일이 생성됨
    - HTML을 열었을 때 주문 수, 판매 수량, 총매출, 평균 주문액 KPI가 보임
    - 카테고리별 매출과 매출 상위 지역 표가 정상적으로 표시됨
    - 반복 실행 시 새로운 이름의 리포트가 계속 생성됨

# 개선사항/느낀점
- 처음에는 데이터 처리와 HTML 생성을 한 파일에 전부 작성하는 것이 간단하다고 생각했지만 역할별로 나누니 각 파일이 무엇을 담당하는지 더 잘 보였음
- config.py를 따로 두면서 데이터 위치나 리포트 제목을 바꿀 때 여러 파일을 수정하지 않아도 된다는 장점을 알게 됨
- Jinja2를 처음 사용할 때는 Python 코드 안에서 HTML을 작성하는 것보다 복잡하게 느껴졌지만, 디자인과 데이터 로직을 분리하니 표나 스타일을 수정하기 쉬웠음
- 파일명에 시간을 넣지 않으면 반복 실행할 때 기존 파일이 계속 덮어써진다는 점을 알게 되었고 자동화에서는 실행 이력을 남기는 것도 중요하다고 느낌
- 현재는 실행 중 오류가 발생하면 다음 실행을 기다리지 못할 수 있으므로 이후에는 try-except와 로그 파일, 재시도 기능을 추가하면 더 안정적일 것 같음
- 실제 운영 환경에서는 sales_raw.csv가 아직 생성되지 않았거나 일부만 저장된 상태일 수 있으므로 파일 존재 여부와 데이터 품질 검사를 더 강화할 필요가 있음
- cron에서는 현재 폴더와 Python 환경이 평소 터미널과 다를 수 있어 절대 경로와 .venv/bin/python을 사용해야 한다는 점이 인상적이었음
- 자동화의 핵심은 단순히 반복문을 만드는 것이 아니라 사람이 직접 실행하지 않아도 같은 과정과 같은 결과가 재현되도록 만드는 것이라고 이해함
- 이후에는 Plotly 차트를 HTML 안에 추가하고 리포트 생성 실패 시 알림을 보내는 기능까지 확장해 보고 싶음
