# 개선사항/느낀점
- df.infor()를 print 함수 내에 넣어 사용하면 마지막에 불필요한 None까지 출력되므로 자체적으로 출력하는 df.info()만 적음
- 반복되는 정제 과정을 함수로 분리하면 코드의 재사용성과 가독성을 높일 수 있음
- 추후에는 결측치 처리, 이상치 처리, 집계, 파일 저장을 각각 별도 함수로 구성하고 자동 테스트도 추가하길 희망
- 현재 제시된 IQR 기준 말고 카테고리마다 다른 기준을 적용하면 더 정교해질 것 같음
- 이번 실습을 통해 데이터 분석에서 바로 집계를 시작하는 것보다 먼저 자료형과 결측치, 이상치를 확인하는 과정이 중요함을 인지함
- 이상치가 포함된 행을 무조건 삭제하면 해당 행의 다른 정상 정보까지 잃게 됨을 이해함
