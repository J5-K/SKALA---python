# Advanced 추가 과제: Sales Pulse - 매출 건강검진 리포트

## 과제 개요

기존 Total3 과제는 `sales_raw.csv`를 정제하고 KPI, 카테고리별 매출, 상위 지역을 HTML 리포트로 자동 생성한다. Advanced 과제에서는 여기서 한 걸음 더 나아가 숫자를 나열하는 데 그치지 않고, **최근 매출에 어떤 변화가 있었는지를 자동으로 발견하고 쉬운 문장으로 설명하는 리포트**를 만든다.

프로그램을 실행하면 최근 30일과 직전 30일의 매출을 비교하고, 카테고리와 지역별 증감률, 할인율별 주문 효과, 평균에서 크게 벗어난 이상 매출일을 분석한다. 분석 결과는 반응형 SVG 차트와 함께 하나의 HTML 파일로 저장된다.

## 이 과제를 떠올리게 된 과정

Total3의 결과를 보면서 "표에 나온 매출 숫자를 본 다음 사용자는 무엇을 해야 할까?"라는 질문에서 시작했다. 기존 리포트는 전체 매출이 얼마인지 보여주지만, 매출이 좋아진 것인지 나빠진 것인지와 어느 부분을 먼저 살펴봐야 하는지는 사용자가 다시 판단해야 한다.

처음에는 고객별 추천이나 이탈 예측도 생각했지만 현재 데이터에는 고객 ID와 고객 행동 이력이 없다. 없는 정보를 억지로 만들기보다 현재 가지고 있는 주문일, 지역, 카테고리, 수량, 가격, 할인율을 충분히 활용하는 편이 더 자연스럽다고 판단했다.

또한 Advanced라는 이름 때문에 복잡한 머신러닝을 사용하는 것보다, 기존에 학습한 Pandas 집계에 간단한 통계 개념을 더하는 것이 적절하다고 생각했다. 최근 기간 비교는 증감률로, 평소와 다른 날짜는 평균과 표준편차를 이용한 z-score로 찾을 수 있다. 구현은 어렵지 않지만 결과는 실제 매출 모니터링 도구처럼 보인다는 점에서 이 프로젝트를 선택했다.

마지막으로 분석 결과를 "매출 12.3% 감소", "가장 성장한 카테고리"처럼 자동 문장으로 만들면 사장님이나 실무자가 표를 모두 읽지 않아도 핵심을 빠르게 파악할 수 있다. 그래서 프로젝트의 콘셉트를 사람의 건강검진처럼 매출 상태를 정상·주의·위험으로 알려주는 **Sales Pulse**로 정했다.

## 주요 기능

1. 데이터 정제
   - 주문일을 날짜 형식으로 변환한다.
   - 수량, 가격, 할인율을 숫자로 변환한다.
   - 잘못된 가격은 같은 카테고리의 중앙값으로 보정한다.
   - 결측 지역과 카테고리는 `Unknown`으로 표시한다.
   - `수량 × 가격 × (1 - 할인율)`로 실제 매출액을 계산한다.

2. 최근 기간 비교
   - 데이터의 마지막 주문일을 기준으로 최근 30일과 직전 30일을 비교한다.
   - 전체, 카테고리, 지역별 매출 증감률을 계산한다.
   - 시스템 날짜가 아니라 데이터의 마지막 날짜를 기준으로 하므로 오래된 실습 데이터에서도 결과가 나온다.

3. 이상 매출일 탐지
   - 일별 매출의 평균과 표준편차를 계산한다.
   - z-score의 절댓값이 2 이상인 날짜를 평소와 크게 다른 날로 표시한다.
   - 복잡한 머신러닝 없이 이해 가능한 통계 기준을 사용한다.

4. 자동 브리핑
   - 전체 매출 변화, 성장 또는 주의 카테고리, 이상 매출일을 자연어 문장으로 만든다.
   - 최근 매출 감소와 이상 징후 수에 따라 정상, 주의, 위험 상태를 표시한다.

5. 시각화
   - SVG로 일별 매출선을 그리고 이상 매출일을 빨간 점으로 강조한다.
   - 반응형 HTML로 만들어 브라우저 크기가 달라져도 확인할 수 있다.

## 파일 구성

```text
Python_data_0720_Day2/
├── Total3/
│   └── sales_raw.csv         # 기존 과제의 원본 데이터
└── Advanced/
    ├── config.py             # 경로와 분석 기준 설정
    ├── sales_analyzer.py     # 정제, 증감률, 이상 탐지, 브리핑 생성
    ├── insight_report.py     # 차트와 HTML 리포트 생성 및 실행
    ├── templates/
    │   └── insight_report.html
    ├── output/               # 실행 결과 HTML 저장 위치
    ├── test_sales_analyzer.py
    └── comm.md
```

## 실행 방법

Total3의 `sales_raw.csv`를 그대로 사용하므로 데이터를 복사할 필요가 없다.

```bash
python Python_data_0720_Day2/Advanced/insight_report.py
```

테스트는 다음과 같이 실행한다.

```bash
pytest Python_data_0720_Day2/Advanced/test_sales_analyzer.py -q
```
