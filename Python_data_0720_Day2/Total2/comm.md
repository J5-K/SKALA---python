# 종합실습 2 EDA + 통계 + ML 파이프라인

종합설명
    - telco_churn.csv의 고객 정보를 이용해서 어떤 고객이 이탈할 가능성이 높은지 확인하는 과제
    - 데이터 확인 → EDA → 통계 검정 → 전처리 → 모델 학습 → 평가 → 결과 저장 순서로 진행
    - 단순히 모델 점수만 확인하는 것이 아니라 실제로 어떤 특성이 이탈과 관련 있는지도 함께 확인함

step0 데이터 확인
    - 전체 데이터는 7,000행, 10열로 구성
    - churn을 예측할 정답으로 사용하고 customer_id는 단순 식별자이므로 모델 입력에서 제외함
    - 처음부터 모델을 만들기보다 데이터 크기와 컬럼, 이탈률을 먼저 확인하도록 구성

step1 Polars EDA
    - 계약 유형과 성별을 기준으로 고객 수와 이탈률을 집계
    - Month-to-month 고객의 이탈률이 약 37%로 One year, Two year보다 높게 나타남
    - Polars를 사용해 집계하고 이후 통계 검정과 scikit-learn 모델링을 위해 pandas DataFrame으로 변환함

step2 통계 검정
    - 이탈 고객과 유지 고객의 월 요금 평균 차이는 t검정으로 확인
    - 계약 유형과 이탈 여부의 연관성은 카이제곱 검정으로 확인
    - t검정 p값은 약 1.226e-20, 카이제곱 p값은 약 1.325e-70으로 모두 0.05보다 작게 나옴
    - 월 요금과 계약 유형이 이탈과 관련 있다는 결과이지만, 이것만으로 직접적인 원인이라고 단정할 수는 없음

step3 전처리와 모델 학습
    - 수치형은 중앙값 대치와 StandardScaler를 적용
    - 범주형은 최빈값 대치와 OneHotEncoder를 적용
    - 전처리와 RandomForest를 하나의 Pipeline으로 연결해서 학습과 예측에 같은 규칙이 사용되도록 함
    - train_test_split에서 stratify를 사용해 학습·평가 데이터의 이탈 비율이 비슷하게 유지되도록 함

step4 평가 및 저장
    - 정확도보다 클래스 구분 능력을 확인하기 위해 ROC-AUC로 평가
    - 실행 결과 ROC-AUC는 약 0.6714로 가이드의 예상값인 약 0.66과 비슷하게 나옴
    - eda_report.html, metrics.json, churn_model.joblib을 output 폴더에 저장
    - Plotly HTML에서 계약별 이탈률과 월 요금 분포를 직접 확인할 수 있음

실행
    - .venv/bin/python Python_data_0720_Day2/Total2/analysis.py

성공 신호
    - 두 통계 검정의 p값이 0.05보다 작음
    - ROC-AUC가 약 0.66 전후로 출력됨
    - output 폴더에 HTML, JSON, joblib 파일이 생성됨

# 개선사항/느낀점
- 처음에는 모델만 실행해서 점수가 나오면 끝이라고 생각했지만, EDA와 통계 검정을 먼저 하니 결과를 설명하기가 훨씬 쉬웠음
- p값이 매우 작다고 해서 해당 변수가 이탈의 직접적인 원인이라고 말할 수 없다는 점을 주의해야겠다고 느낌
- 전처리 코드를 모델과 따로 관리하면 학습할 때와 예측할 때 처리 방식이 달라질 수 있는데 Pipeline으로 묶으면 이런 실수를 줄일 수 있다는 점이 유용했음
- customer_id와 churn을 입력 데이터에서 제외하는 과정을 통해 데이터 누수가 모델 성능을 잘못 높일 수 있다는 점을 알게 됨
- 현재는 RandomForest 한 모델만 사용했으므로 이후에는 LogisticRegression이나 GradientBoosting과 ROC-AUC를 비교해 보고 싶음
- ROC-AUC만으로 모델이 충분히 좋다고 판단하기보다 정밀도, 재현율, 혼동행렬도 함께 확인하면 이탈 고객을 놓치는 비율을 더 구체적으로 알 수 있을 것 같음
- Plotly 결과를 HTML로 저장하니 코드 실행 결과를 표로만 보는 것보다 계약 유형별 차이가 잘 보여서 분석 내용을 전달하기 쉬웠음
- 아직 통계 검정 결과와 모델 성능을 자연스럽게 해석하는 부분이 익숙하지 않아 단순히 숫자만 적지 않고 의미를 설명하는 연습이 더 필요함
