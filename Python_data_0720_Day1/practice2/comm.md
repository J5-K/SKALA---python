# Pydantic v2 중첩 스키마 검증

step0 오염 요소 찾기
    - 키와 값 타입을 훑고 어떤 오류일지 의심하고 예상

step1-3 단순한 모델 만들고 범위 및 제약조건 확장하기
    - 타입, 범위 등 제약 조건을 설정하고 Field()로 표현
    - 커스텀 규칙: Field()로 표현 안되는 (소문자, 이메일에 @포함)은 직접 함수로 씀

step4-5 중첩구조 - 모델 안 모델
    - json 파일을 참조하니 Profile 내에 다시 여러 값이 있는 것을 확인함
    - from typing import Literal과 Basemodel을 활용해 profile 모델을 따로 만들고 바깥 함수에도 추가해줌

#개선사항/느낀점
    - 오염 요소를 고민할 때 먼저 음수 값/타입 이상/필수 필드 누락/범위초과를 의심하고 해당 조건들을 필터링하며 빠르게 오류를 바로잡을 수 있음
    - 커스텀규칙을 적용하는 문법에 대해 익숙해져야 할 필요가 있음
    - 단순히 코드를 걸러내는 것 말고 실패해도 멈추지 않는 구조(try/except)와 "왜 실패했는지" 남기는 것의 중요성을 인식함
