# asyncio 기반 비동기 수집기

step0 동기 버전 6.04초, 결과 60건, 오류없음

step1 비동기 버전(async/await) 6.56초, 결과 60건, 오류없음
    - async: def 앞/ await: 기다리는 곳 앞
    - time.sleep()은 async 함수 안에 쓰면 안됨 프로그램 전체 멈춤 -- asyncio.run()이용하기

step2 비동기 버전 0.10초, 결과 60건, 오류없음
    - 60개 무제한으로 던진 상황- 실무에선 주의해야 함

step3 비동기 버전 0.6초, 결과 60건, 오류없음
    - 백프레셔 Semaphore로 동시 요청 수를 제한해 동시 요청을 MAX_CURRENT로 묶어주는 안전장치
    - 필요도: 제한이 없을 경우 서버 과부하나 IP 차단 등의 문제 발생 가능

step4-6 타임아웃/재시도/예외격리
    - 20, 40, 60번은 첫 시도에서 연결 오류가 발생하도록 하고 두 번째부턴 정상 작동
    - 25, 50번은 첫 시도에서 타임아웃이 발생하도록 함 TIMEOUT_SECONDS보다 오래 기다리기 때문에 타임아웃
    - 인터넷 없이 실행하는 모의 API 모드 - USE_REAL_HTTP = False
    - 정상적인 API 응답 대기 시간을 흉내 - await asyncio.sleep(REQUEST_DELAY)
    - 동시에 최대 10건만 요청/마지막 시도까지 실패했다면 더 이상 재시도하지 않음
    - 요청을 최대 0.3초까지만 기다림/일부 작업이 최종 실패하더라도 전체 작업을 계속

# 개선사항/느낀점
    - step2에서 60건을 한번에 시작해 매우 빠르지만 실제 상황에서는 서버에 부담을 줄 수 있음 
    - semaphore을 유용하게 활용할 수 있어야 할 듯
    - 함수를 활용해 오류를 직접 발생시키고 예외를 격리시키는 과정에서 다양한 방법을 활용해볼 수 있겠다는 생각이 듦
