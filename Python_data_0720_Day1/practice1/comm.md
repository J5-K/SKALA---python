# 대용량 로그 스트리밍 집계

step2-3 Counter함수 vs defaultdict(Counter() 선택)
    - Counter존재하지 않는 키 조회하면 0으로 처리해서 바로 증가시킬 수 있음
    - 사용법과 거의 같지만 Counter는 횟수 집계 전용이라 most_common()기능 제공 장점 있음
    - 코드의 목적이 접속 횟수 세는 것이므로 Counter가 더 자연스럽다고 판단
    - 경로별, 시간대별, IP별 지표를 늘려 Counter함수를 이용해 지표를 늘리는 과정에서 처음에는 변수에 따로 저장하지 않고 아래와 같은 형식으로 구성해 코드가 길고 읽기 어려웠으나 필요한 값을 꺼내놓고 아래에서 짧게 사용할 수 있도록 현재 버전으로 수정
    status_counts[log["status"]] += 1
    path_counts[log["path"]] += 1
    ip_counts[log["ip"]] += 1

step4-5 soulution vs solution_backup(for문)
    - 출력물은 비슷하지만 집계하는 코드 구조 즉, 어떤 방식으로 계산하는지가 바뀜
    - 기존 solution.backup에 있는 파일은 for문 안에서 직접 누적한 반면 최종본에서는 accumulate log() 함수로 분리하고 reduce()가 모든 로그에 그 함수를 반복 적용함
    - 이전에는 읽기, 집계, 출력이 main() 안에 섞여있어 무거웠다면 지금은 역할이 분리됨
    - 하나의 result 안에 들어가서 나중에 다른 함수로 넘기거나 파일로 저장하기가 쉬워짐
    - 공통점은 모두 csv를 한 번만 읽고 한 행씩 처리

개선사항/느낀점
    - 아직은 어떤 코드가 더 간단한 것인지 판단하는 데 혼자만의 힘으로는 구별이 어려움
    - 코드의 간결성에 중점을 두려고 하다보니 요구사항을 구현하는 데 시간이 더 많이 소요되는 것 같음 - 중간지점을 찾을 필요가 있을 듯
    - Counter함수와 defaultdict, yield 등 기본적인 용어 숙지가 아직은 미숙해 더 연습이 필요햠
