# 종합실습 1 - 비동기 ETL 파이프라인

- 종합설명 : 실습 2의 Pydantic 검증과 실습 3의 비동기 수집을 하나로 연결함
        - JSON 데이터를 비동기로 수집(Extract), Pydantic으로 검증(Transform), 정상/오류 데이터 분리, CSV & Parquet 저장(Load), pytest로 검증

- 구현 내용:
    비동기 데이터 60건 수집
    동시 요청 최대 10건 제한
    일시적 실패 시 최대 3번 재시도
    Pydantic으로 정상·오염 데이터 분리
    카테고리 공백 제거 및 소문자 변환
    음수 가격 3건 거부
    정상 데이터 CSV·Parquet 저장
    pytest 테스트 6개 포함

- 먼저 60건의 상품 데이터를 동시에 수집하되 Semaphore로 동시 요청을 10건까지 제한했고, 일시적인 오류가 발생하면 다시 시도하도록 만듦

- 수집한 데이터는 Pydantic으로 검증하여 정상 데이터와 오염 데이터를 분리

- 카테고리는 소문자로 정리하고 가격이 0보다 작으면 오염 데이터로 처리

- 정상 데이터만 pandas DataFrame으로 만든 뒤 CSV와 Parquet 파일로 저장

# 리뷰 
- 처음에는 한 함수에 전부 작성하는 것이 편해 보였지만, Extract·Transform·Load로 나누니 각 부분의 역할이 훨씬 잘 보임

- pytest 테스트 6개를 작성하면서 저장 결과와 데이터 검증 규칙도 함께 확인

- 실행 결과는 전체 60건 중 정상 57건, 오염 3건이며 정상 데이터 57건이 저장된다.
